Caesar Cipher Basics
A Caesar cipher shifts each letter by a fixed number (the "key"). For example, with shift 3: A→D, B→E, etc.
Formula (Encrypt)
C = (P + K) mod 26
Formula (Decrypt)
P = (C - K + 26) mod 26
Key Space
25 possible keys
Security
⚠ Not secure
Attack Methods:

1. Brute Force: Try all 25 possible shifts (shown in the Cipher tab).

2. Frequency Analysis: Analyze letter frequencies to find the most likely shift (Analysis tab).

3. Known Plaintext: If you know any word from the plaintext and its ciphertext equivalent, you can deduce the shift immediately.
Why Caesar Cipher is Insecure:

• Only 25 possible keys (trivially brute-forceable)
• Letter frequency patterns remain visible after encryption
• No authentication or integrity checking
• Breaks in seconds on modern computers
Common Use Cases Today:

• Educational purposes
• CTF (Capture The Flag) challenges
• Simple obfuscation (not real security)
• Historical/archaeological text analysis
